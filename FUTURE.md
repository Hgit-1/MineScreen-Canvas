# MineScreen Future Roadmap

WebDisplays behavior parity is tracked in `PORTING.md`. This file covers optional features
beyond the original mod rather than replacing the port matrix.

本文档记录尚未进入当前稳定功能集的长期方向。条目代表设计预期，不承诺具体版本或发布日期。

## 新设备与便携终端

### 手机

- 可手持的MineScreen手机，提供小尺寸触控屏、网页、媒体遥控和屏幕投送入口。
- 手机可作为主机/屏幕的便携遥控器，也可在获得权限后预览指定 `screen_id`。
- 为多人环境设计独立的本地凭据仓库；不会把Cookie、密码、VNC凭据或本地媒体路径同步到服务器。
- 研究物品模型上的实时低分辨率渲染，并提供帧率、距离和电量/发热风格的性能限制。

### 行车导航仪

- 可放置或安装在载具上的导航终端，显示地图、路线、速度、航向和媒体控制。
- 与常见载具Mod通过可选适配器集成，不强制依赖某一种车辆API。
- 支持触控式大按钮、夜间主题、驾驶期间输入限制以及服务器权限校验。
- 后续研究网页地图、离线地图瓦片与原生Minecraft地图数据之间的切换。

## 屏幕与光照

### 视频驱动光照更新

- 从VIDEO/WEB/VNC画面计算低频平均亮度、主色和边缘亮度，用于屏幕附近的环境光变化。
- 默认只使用降采样亮度网格，不读取或同步完整视频帧。
- 光照更新频率与画面帧率分离，预计限制在1–5Hz，并设置变化阈值避免区块光照频繁重建。
- 优先研究客户端光学/着色器发光，不直接修改服务端方块光照；服务端动态光作为可选高成本模式。
- 为光敏感用户提供亮度上限、色温、闪烁抑制和完全关闭选项。
- 与动态光源、光影包和渲染优化Mod通过可选适配器协作。

## 分发与同步

- 视频点播使用内容哈希、分片缓存和可选P2P交换，游戏服务器只同步manifest与时间轴。
- 实时WEB已提供“权威渲染节点 + JPEG + P2P树形转发”MVP；后续升级为H.264/AV1硬件编码、WebRTC/QUIC、NAT穿透、端到端加密和动态树修复。
- 增加媒体文件大小、时长和内容哈希验证，确保同一 `media_id` 映射到一致文件。
- 改进服务器时钟偏差估计、RTT补偿和小幅播放速率校正，减少视频硬seek。
- P2P不可用时始终保留客户端直连内容源的安全回退。

## UI与可访问性

- 继续扩展 `MineScreenUiProvider`，允许Otyacraft Engine Renewed等UI框架通过适配器接管外观。
- Cloth Config API适配器用于更完整的配置页面；Architectury API适配器用于跨加载器UI集成层。
- 这些前置并非彼此通用的控件皮肤API，因此采用独立适配模块，不在核心Mod中硬编码其内部类。
- 增加触控布局、键盘导航、屏幕阅读器提示、颜色对比预设和可缩放控件密度。

## VNC

- 在当前Tight支持基础上评估ZRLE、H.264扩展编码和硬件视频解码。
- 增加吞吐量、压缩率、更新区域和解码耗时统计面板。
- 多人观看同一VNC时复用现有P2P树，提供可选“单一RFB上游 + Tight脏矩形/编码矩形直转”模式，减少VNC服务器重复连接和上行；控制事件沿树回传。
- 研究VeNCrypt/TLS，替代经典RFB DES验证和明文传输。
